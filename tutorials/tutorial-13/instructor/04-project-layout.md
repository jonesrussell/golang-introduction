# Instructor Notes: Project Layout

## Teaching Techniques
- Show standard project layout
- Demonstrate organization patterns
- Show common directories
- Emphasize: "Good layout makes code maintainable"

## Key Emphasis
- **cmd/**: Main applications
- **internal/**: Private packages
- **pkg/**: Public packages
- **Standard layout**: Follow conventions

## Common Questions
- "How do I organize my project?" - Follow standard layout
- "What goes in internal/?" - Private packages
- "What goes in pkg/?" - Public packages

## Engagement
- "Notice how this layout organizes code"
- "Standard layout makes projects familiar"
- "This is how Go projects are organized"

## Real-World Context
- Standard layout is common
- Makes projects familiar
- Used in many Go projects
