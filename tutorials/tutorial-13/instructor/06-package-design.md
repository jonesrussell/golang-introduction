# Instructor Notes: Package Design

## Teaching Techniques
- Show good package design
- Demonstrate cohesion
- Show what belongs together
- Emphasize: "Packages should be cohesive"

## Key Emphasis
- **Cohesion**: Related functionality together
- **Size**: Not too big, not too small
- **Dependencies**: Minimize dependencies
- **API**: Clear, focused API

## Common Questions
- "How big should a package be?" - Cohesive, focused
- "What belongs together?" - Related functionality
- "How do I design a package?" - Start with functionality

## Engagement
- "Notice how these packages are cohesive"
- "Each package has a clear purpose"
- "Good design makes code maintainable"

## Real-World Context
- Good package design is important
- Makes code maintainable
- Standard practices exist

## Transition
- "Let's put it all together in a practical example..."
