# Instructor Notes: User Experience

## Teaching Techniques
- Show UX best practices
- Demonstrate help text
- Show error messages
- Emphasize: "Good UX makes tools usable"

## Key Emphasis
- **Help text**: Clear, concise, examples
- **Error messages**: Helpful, actionable
- **Output**: Clear, formatted
- **Validation**: Validate inputs early

## Common Questions
- "How do I write good help?" - Clear, with examples
- "What about error messages?" - Helpful, actionable
- "How do I format output?" - Use tables, colors, etc.

## Engagement
- "Notice how good help makes tools usable"
- "Error messages should help users"
- "UX matters for CLI tools"

## Real-World Context
- Good UX is important
- Makes tools more usable
- Standard practices exist

## Transition
- "Let's see how to test CLI tools..."
