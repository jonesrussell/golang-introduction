# Instructor Notes: Introduction

## Teaching Techniques
- Show enthusiasm for structured logging
- Contrast printf vs structured logging
- Emphasize: "Structured logging enables better observability"
- Preview the production logging setup

## Key Points to Emphasize
- Structured logging is essential for production
- Printf is hard to parse and search
- Zap is fast and structured
- Logging is critical for debugging

## Engagement
- Ask: "How do you debug production issues?"
- Mention: "Structured logging makes debugging easier"
- Promise: "You'll set up production-ready logging"

## Real-World Context
- Structured logging is standard in production
- Essential for observability
- Zap is widely used
